development and CI only. For production, install cert-manager and set tls.provider=certManager.

   webhook pods are NOT automatically restarted. After upgrading, run:
     kubectl rollout restart deployment/{{ include "chur.fullname" . }}-webhook -n {{ .Release.Namespace }}

   is handled {{ if eq .Values.tls.provider "certManager" }}by cert-manager{{ else }}by replacing the user-provided Secret{{ end }}.
   The webhook deployment does not automatically restart when the TLS certificate
   is rotated — you may need to manually restart the pods or use a tool like
   stakater/reloader.
{{- end }}

To inject secrets into a Pod, annotate it:

  apiVersion: v1
  kind: Pod
  metadata:
    annotations:
      chur.io/provider: "k8s"
      chur.io/secret-ref: "my-secret"
  spec:
    serviceAccountName: {{ .Values.rbac.serviceAccountName }}
    containers:
      - name: app
        image: my-app
